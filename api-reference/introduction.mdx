# Mdadda API Overview

The Mdadda API is a comprehensive REST API that powers Kenya's premier house helper marketplace platform. Our API enables developers to build applications that connect house helpers with households through sophisticated matching algorithms, real-time communication, and a secure marketplace system.

## Platform Architecture

Mdadda consists of a Django REST Framework backend with real-time capabilities, providing:

- **Comprehensive User Management**: Authentication, profiles, and role-based access
- **Marketplace System**: Token-based economy with subscriptions and promotions
- **Intelligent Matching**: AI-powered algorithms connecting helpers with households
- **Real-time Communication**: WebSocket-based chat with typing indicators
- **Verification System**: Identity and background verification for trust and safety
- **Location Services**: Kenya-focused geocoding and location management

## Base URL & Environment

```
Production: https://api.mdadda.com/api/v1/
Staging: https://staging-api.mdadda.com/api/v1/
```

All API endpoints are served over HTTPS and follow RESTful conventions.

## Authentication

### Firebase JWT Authentication

Mdadda uses Firebase Authentication as the primary authentication method. All API requests must include a valid Firebase JWT token in the Authorization header.

```http
Authorization: Bearer <firebase_jwt_token>
```

### Authentication Flow

<Steps>
  <Step title="Firebase Authentication">
    Authenticate users through Firebase Auth using email/password, Google, or other supported providers
  </Step>
  <Step title="Token Exchange">
    Obtain the Firebase ID token from your authenticated Firebase user
  </Step>
  <Step title="API Request">
    Include the token in the Authorization header for all API calls
  </Step>
</Steps>

### Example Authentication

<CodeGroup>
```javascript JavaScript
import { getAuth, signInWithEmailAndPassword } from 'firebase/auth';

const auth = getAuth();
const userCredential = await signInWithEmailAndPassword(auth, email, password);
const token = await userCredential.user.getIdToken();

// Use token in API calls
const response = await fetch('https://api.mdadda.com/api/v1/users/profile/', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});
```

```python Python
import requests
import firebase_admin
from firebase_admin import auth

# Initialize Firebase Admin SDK
cred = firebase_admin.credentials.Certificate('path/to/serviceAccountKey.json')
firebase_admin.initialize_app(cred)

# Verify token and make API call
def api_request_with_auth(token, endpoint):
    try:
        # Verify the token
        decoded_token = auth.verify_id_token(token)

        headers = {
            'Authorization': f'Bearer {token}',
            'Content-Type': 'application/json'
        }

        response = requests.get(f'https://api.mdadda.com/api/v1/{endpoint}', headers=headers)
        return response.json()
    except Exception as e:
        print(f'Authentication error: {e}')
```

```curl cURL
curl -X GET "https://api.mdadda.com/api/v1/users/profile/" \
  -H "Authorization: Bearer YOUR_FIREBASE_JWT_TOKEN" \
  -H "Content-Type: application/json"
```

## Rate Limiting

To ensure fair usage and platform stability, the Mdadda API implements rate limiting:

- **Standard endpoints**: 1000 requests per hour per user
- **Authentication endpoints**: 60 requests per hour per IP
- **Chat endpoints**: 500 messages per hour per user
- **Search endpoints**: 100 requests per hour per user

Rate limit headers are included in all responses:

```http
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1642251600
```

## WebSocket Connections

For real-time features like chat, Mdadda provides WebSocket connections:

```
Production: wss://api.mdadda.com/ws/
Staging: wss://staging-api.mdadda.com/ws/
```

### WebSocket Authentication

Include the Firebase JWT token in the connection query parameters:

```javascript
const ws = new WebSocket(`wss://api.mdadda.com/ws/chat/${conversationId}/?token=${firebaseToken}`);
```

### Quick Start Example

<CodeGroup>
```javascript JavaScript
// 1. Initialize Firebase and authenticate
import { initializeApp } from 'firebase/app';
import { getAuth, signInWithEmailAndPassword } from 'firebase/auth';

const app = initializeApp(firebaseConfig);
const auth = getAuth();

// 2. Sign in and get token
const userCredential = await signInWithEmailAndPassword(auth, email, password);
const token = await userCredential.user.getIdToken();

// 3. Make your first API call
const response = await fetch('https://api.mdadda.com/api/v1/users/profile/', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }

const userData = await response.json();
console.log('User profile:', userData);
```

```python Python
import requests
import firebase_admin
from firebase_admin import auth

# 1. Initialize Firebase Admin
cred = firebase_admin.credentials.Certificate('serviceAccountKey.json')
firebase_admin.initialize_app(cred)

# 2. Create a custom token for testing
uid = 'user123'
custom_token = auth.create_custom_token(uid)

# 3. Make API call
headers = {
    'Authorization': f'Bearer {custom_token.decode()}',
    'Content-Type': 'application/json'
}

response = requests.get('https://api.mdadda.com/api/v1/users/profile/', headers=headers)
user_data = response.json()
print('User profile:', user_data)
```
